# 文件树ToStr 架构规范

## 1. 项目概述

"文件树ToStr" 是一款 Windows 桌面工具，用于扫描指定目录，构建文件树结构，收集文本文件内容，可选通过 TrID 识别文件类型，最终输出为 JSON 文件。

### 核心功能

- 递归扫描目录，构建完整文件树（含文件大小、修改时间）
- 按扩展名勾选需要收集文本内容的文件
- 支持无扩展名文件的内容收集
- 可选 TrID 格式识别（基于文件二进制特征）
- 启动时异步扫描目录，先显示加载界面
- 配置持久化（JSON 配置文件）
- 结果 JSON 输出

### 技术栈

| 层级 | 技术 |
|------|------|
| 语言 | Python 3.11 |
| GUI | Tkinter (ttk) |
| 文件类型识别 | TrID (lib/trid_lite.py) |
| 配置存储 | JSON |
| 并发 | threading + ThreadPoolExecutor (后台扫描 / 文件处理) |

### 设计原则

- **主源码不使用 `__init__.py`**：项目通过 `run.py` 中的 `sys.path.insert(0, 'src')` 将 `src/` 加入搜索路径，所有子目录作为命名空间包直接导入，无需包声明。
- **测试目录使用 `__init__.py`**：`test/` 及其子目录使用 `__init__.py` 满足 pytest 包发现需求。
- **单向依赖**：`core/` → 无依赖；`ui/` → 依赖 `core/`、`data/`、`lib/`；`main.py` → 依赖 `ui/`。
- **高内聚低耦合**：每个模块职责单一，功能边界清晰。
- **事件驱动**：后台线程完成任务后通过 `root.after(0, callback)` 回调主线程，消除轮询。

## 3. 模块详解

### 3.1 根入口 `run.py` (21行)

**职责**：启动前的环境准备，然后委托给 `main.py`。

- 计算 `src/` 路径并加入 `sys.path`
- 调用 Win32 API 最小化控制台窗口（`SW_MINIMIZE`）
- 导入并调用 `main.main()`

### 3.2 程序入口 `src/main.py` (30行)

**职责**：日志初始化、命令行参数解析、启动主窗口。

- 配置 `logging`（INFO 级别，时间戳格式）
- 解析命令行参数：可选指定配置文件路径
- 创建 `ConfigWindow` 实例并调用 `run()`

**对外接口**：

| 函数 | 说明 |
|------|------|
| `main()` | 程序入口，配置日志、解析参数、启动窗口 |

### 3.3 核心层 `src/core/`

#### 3.3.1 `tree.py` —— 目录树构建 (128行)

**职责**：在预构建的轻量目录树上，使用 `ThreadPoolExecutor` 并行填充文件内容与 TrID 分析结果。

**对外接口**：

| 函数 | 说明 |
|------|------|
| `format_mtime(timestamp)` | 时间戳 → ISO 8601 字符串 |
| `is_auto_generated_file(filepath)` | 判断文件是否为本工具自动生成的（读取并验证 JSON 结构） |
| `fill_tree(tree, extensions, TDB, skip_trid_paths, no_ext_paths, root_path)` | 遍历预构建树，多线程读取文件内容 + TrID 分析，填充到树节点中 |

**内部函数**：

| 函数 | 说明 |
|------|------|
| `_process_file(full_path, fname, extensions, TDB, skip_trid_paths, no_ext_paths, current_rel)` | 在线程中处理单个文件：TrID 分析 + 内容读取 + 无扩展名内容注入 |

**数据结构**：

```python
TreeDict = {
    "files": {"文件名": {"content": "文本内容或None", "size": 字节数, "mtime": "ISO时间", "trid": [...]}},
    "dirs":  {"子目录名": TreeDict}
}
```

**处理流程**：边遍历树边提交线程池 → 按序遍历树等待 future 完成 → 写入结果到树节点。工作线程数 = CPU 核心数。

**无扩展名文件内容注入**：从 `_inject_no_ext_content()` 重构为内联到 `_process_file()` 中，基于 `no_ext_paths` 集合匹配，与扩展名文件统一处理。

#### 3.3.2 `trid.py` —— TrID 集成 (67行)

**职责**：TrID 定义包查找、加载、可用性检测。

**对外接口**：

| 变量/函数 | 说明 |
|------|------|
| `TRID_AVAILABLE` | `bool`，trid_lite 模块是否可用 |
| `tridAnalyze` | 函数，文件格式分析（来自 trid_lite） |
| `find_triddefs()` | 按优先级搜索 `triddefs.trd` 路径 |
| `load_tdb(triddefs_path)` | 加载 TrID 定义包，返回 `TrIDDefsBlock` 对象（含缓存） |

**搜索优先级**：当前目录 → 上级目录 → 子目录 → 脚本目录 → `data/` → 所有固定磁盘根目录（仅 Win32）

#### 3.3.3 `scanner.py` —— 文件系统扫描 (297行)

**职责**：BFS 多线程遍历目录，并行 stat 所有文件，构建轻量目录树。

**对外接口**：

| 函数 | 说明 |
|------|------|
| `load_text_mime_extensions()` | 从标准库 `mimetypes` + 补充列表加载所有文本扩展名集合 |
| `scan_directory(root_path, exclude_names)` | BFS + 线程池扫描：返回 (排序后的扩展名列表, 无扩展名文件信息列表, 轻量目录树) |

**内部函数**：

| 函数 | 说明 |
|------|------|
| `_is_text_mime(mime)` | 判断 MIME 类型是否为文本类型（含 `+xml`/`+json` 后缀匹配） |
| `_stat_file(full_path)` | 获取文件 stat 信息 |
| `_scandir_one(dirpath, rel_dir)` | 在线程中 scandir 单个目录 |

**算法**：

- 阶段 1：BFS + 线程池，每个线程 `scandir` 一个目录，一批结束后批量提交子目录
- 阶段 2：多线程并行 `stat` 所有已发现文件
- 阶段 3：单线程根据结果构建轻量目录树（`files` 中仅含 `path`/`size`/`mtime`，不含 `content`/`trid`）

#### 3.3.4 `pipeline.py` —— 处理流水线 (76行)

**职责**：串联 TrID 加载、目录树填充、JSON 输出。

**对外接口**：

| 函数 | 说明 |
|------|------|
| `process_request(root_path, extensions, exclude_names, use_trid, selected_no_ext, output_path, file_tree)` | 主处理流程：加载 TrID → 深拷贝树 → 填充内容 → 输出JSON |

**变更说明**：不再负责 `build_tree()` 和 `_inject_no_ext_content()`，这些职责分别移入 `scanner.py`（树构建）和 `tree.py`（内容注入）。

### 3.4 界面层 `src/ui/`

#### 3.4.1 `app.py` —— 主窗口 (322行)

**职责**：Tkinter `ConfigWindow` 类，管理整个配置窗口的 UI 和交互。

**ConfigWindow 类**：

| 方法 | 说明 |
|------|------|

| `_start_initial_scan()` | 后台线程：扫描目录 + 加载配置 + 自动勾选文本扩展名 + 回调主线程 |
| `_on_scan_complete()` | 扫描完成后回调（主线程）：销毁加载界面，构建主 UI |
| `_on_window_close()` | 窗口关闭协议：先恢复光标再销毁窗口 |
| `_build_ui()` | 构建完整界面：扩展名列表、无扩展名区、TrID选项、按钮 |
| `_calc_columns()` | 用 `math.isqrt` 近似计算多列扩展名复选框的最优列数 |
| `_calc_window_size()` | 基于扩展名数量动态计算窗口尺寸 |
| `_center_window()` | 窗口居中 |
| `_build_extension_list()` | 构建扩展名复选框网格 |
| `_build_no_ext_section()` | 构建无扩展名文件按钮区 |
| `_open_no_ext_dialog()` | 打开 `NoExtFileDialog` 弹窗 |
| `_build_trid_option()` | 构建 TrID 选项复选框 |
| `_build_buttons()` | 构建确认按钮 |
| `_on_confirm()` | 用户点击"确定"：保存配置 → 确认覆盖 → 启动后台处理线程 |
| `_on_process_complete()` | 处理完成后回调（主线程）：显示结果 |
| `_show_result()` | 显示处理结果（成功/失败） |
| `_on_save_as()` | 结果另存为功能 |
| `run()` | 启动主事件循环 |

**启动流程**：

```
ConfigWindow.__init__()
  → 创建窗口、置顶、显示加载界面
  → _start_initial_scan() 启动后台线程
      ├─ set_busy_cursor()
      ├─ scan_directory()        → 扫描目录，构建轻量树
      ├─ load_config() + apply_config()  → 恢复配置
      │   （若无配置则自动勾选纯文本扩展名）
      ├─ restore_system_cursor()
      └─ root.after(0, _on_scan_complete)
```

**依赖关系**：

```
ConfigWindow
  ├── core/scanner  → scan_directory, load_text_mime_extensions
  ├── core/pipeline → process_request
  ├── core/tree     → TreeDict
  ├── core/trid     → TRID_AVAILABLE
  ├── ui/config     → EXCLUDE_NAMES_DEFAULT, load_config, save_config, apply_config
  ├── ui/cursor     → set_busy_cursor, restore_system_cursor
  └── ui/dialogs    → NoExtFileDialog
```

#### 3.4.2 `config.py` —— 配置管理 (74行)

**职责**：配置文件（JSON）的加载、保存、应用。

**对外接口**：

| 常量 | 说明 |
|------|------|
| `EXCLUDE_NAMES_DEFAULT` | 默认排除文件名集合（含 `triddefs.trd`、`.triddefs.trd.cache`） |
| `CONFIG_FILENAME` | 配置文件名 `tree_to_str-config.json` |

| 函数 | 说明 |
|------|------|
| `load_config(config_path)` | 加载 JSON 配置文件 |
| `save_config(config_path, extensions, no_ext_files, use_trid)` | 保存配置到 JSON |
| `apply_config(cfg, all_extensions, no_ext_files)` | 将配置应用到当前扫描结果，过滤不存在的条目 |

**配置文件格式** (`tree_to_str-config.json`)：

```json
{
  "extensions": [".py", ".txt", ".json"],
  "no_ext_files": ["path/to/file"],
  "use_trid": true
}
```

#### 3.4.3 `cursor.py` —— 系统光标管理 (126行)

**职责**：通过 Win32 API 管理系统级光标忙状态，并在进程退出时自动恢复。

**对外接口**：

| 函数 | 说明 |
|------|------|
| `set_busy_cursor()` | 通过 SetSystemCursor 将系统默认光标替换为忙光标（转圈），返回是否成功 |
| `restore_system_cursor()` | 通过 SystemParametersInfoW(SPI_SETCURSORS) 恢复系统默认光标 |

**模块导入时自动注册的清理机制**：

| 机制 | 覆盖场景 |
|------|------|
| `atexit` | 正常退出、sys.exit()、未捕获异常 |
| `signal.Signals` 遍历 | 所有平台支持的终止信号（SIGINT、SIGTERM、SIGBREAK、SIGABRT、SIGILL、SIGFPE、SIGSEGV） |
| `SetUnhandledExceptionFilter` | C 扩展/解释器级崩溃（访问冲突、除零等） |
| `SetConsoleCtrlHandler` | 关闭命令行窗口（CTRL_CLOSE_EVENT）、注销/关机 |

**注意**：`SetSystemCursor` 是系统级替换，进程异常退出时光标不会自动恢复，因此本模块在导入时即注册所有可用的退出清理机制，确保光标在任何退出场景下都能恢复。

#### 3.4.4 `dialogs.py` —— 对话框组件 (464行)

**职责**：提供无扩展名文件弹窗及搜索组件。

| 类 | 说明 |
|------|------|
| `FileSearch` | Everything-like 搜索语法解析器（支持 AND/OR/NOT、通配符、size/dm/regex 等函数） |
| `NoExtFileDialog` | 无扩展名文件查看与选择弹窗（Treeview 列表，支持勾选/搜索/排序/置顶） |

### 3.5 第三方库 `src/lib/`

#### `trid_lite.py` (191行)

**职责**：TrID 文件类型识别引擎的 Python 移植版，解析 `.trd` 定义包，对文件进行二进制模式匹配。

**核心类**：

| 类 | 说明 |
|------|------|
| `TrIDDef` | 单个文件类型定义（模式、字符串、扩展名等） |
| `TrIDResult` | 单次匹配结果（百分比、得分、模式数、字符串数） |
| `TrIDDefsBlock` | 定义包容器（版本、定义数、分组） |

**核心函数**：

| 函数 | 说明 |
|------|------|
| `trdpkg2defs(filename, usecache)` | 解析 `.trd` 文件，返回 `TrIDDefsBlock` |
| `tridAnalyze(filename, TDB)` | 分析单个文件，返回 `TrIDResult` 列表 |

### 3.6 静态数据 `src/data/`

#### `triddefs.trd`

TrID 二进制定义包，包含数千种文件格式的识别规则。

#### `.triddefs.trd.cache`

由 `trid_lite.trdpkg2defs(usecache=True)` 自动生成的解析缓存文件，加速后续加载。

```

  │
  ├─ sys.path 配置

                                │
                                └─ [用户确认] _on_confirm()
                                     │
                                     ├─ config.save_config()   → 持久化配置
                                     │
                                     └─ [后台线程] pipeline.process_request()

                                          ├─ trid.find_triddefs()      → 查找 TrID 定义包

                                               │
                                               └─ root.after(0, _on_process_complete)
                                                    └─ 显示结果（成功/失败）+ 另存为按钮
```
